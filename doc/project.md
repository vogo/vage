# 项目总览:vage

## 愿景

为 Go 生态提供一个**可组合、可观测、可运维**的 LLM 智能代理框架,让开发者用统一的抽象组装从单个工具调用型 Agent 到复杂 DAG 多代理工作流的完整系统,而无需绑定任何单一大模型厂商。

## 我们要解决的问题

裸调用大模型 SDK 构建 Agent 时,开发者被迫反复自造同一批基础设施:ReAct 循环、工具调度、多轮记忆与上下文压缩、token 预算、安全护栏、断点续跑、可观测事件流、多代理编排。vage 把这些沉淀为**互相解耦、可独立替换的构件**,并用供应商中立的数据契约(`schema`)把它们粘合起来。

## 范围

**做:**

- 智能体抽象:任务型(ReAct)、路由型、工作流型(DAG)、自定义型四种一等形态。
- DAG 编排:并行、循环、条件、补偿(Saga)、检查点、背压、优先级调度。
- 三层记忆:working(请求)→ session(对话)→ store(持久),含上下文压缩与 token 预算。
- 能力接入:LLM 中间件链、本地/MCP/agent-as-tool 工具、Agent Skills。
- 安全:输入/输出/工具结果护栏、凭证脱敏。
- 运行时:HTTP 服务(sync/streaming/async)、hook 事件总线、评测器、向量召回。

**非目标(不做):**

- 不自研大模型推理或托管,通过 `github.com/vogo/aimodel` 接入外部模型 SDK。
- 不做前端 UI、不做可视化编排器。
- 不做通用工作流引擎;编排能力聚焦于"驱动 Agent 与工具"这一场景。
- 不内置向量数据库,只定义可插拔的向量召回接口(自带内存实现,qdrant 等为外部后端)。
- 不做多租户账户体系、计费、配额中心(留给使用方)。

## 相关方

| 角色 | 关注点 |
|------|--------|
| 框架使用者(Go 开发者) | 组装 Agent、接入自有模型与工具、可观测与可运维 |
| 维护者(vogo 组织) | API 稳定性、供应商中立、契约层向后兼容 |
| 安全/合规负责人 | 护栏有效性、凭证不泄露、工具调用可审计 |

## 交付形态

- **Go 库**:`go get github.com/vogo/vage`,以包为单位按需引入。
- **HTTP 服务**:`service` 包提供 REST 端点,可独立部署为代理运行时。
- **MCP server**:通过 `mcp/server` 把 Agent 能力作为 MCP 工具对外暴露。

## 当前状态

- 语言/工具链:Go 1.26,Apache-2.0 许可。
- 质量门禁:`make build` = license-check → format → lint → test;覆盖率上报 Codecov。
- 代码规模量级:约 400 个 Go 源文件、200+ 测试文件;`integrations/` 下有跨包集成测试矩阵。
- 阶段:活跃开发中,依赖 `github.com/vogo/aimodel@v0.4.0`。
